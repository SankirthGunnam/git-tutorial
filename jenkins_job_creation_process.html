Jenkins Job Creation Process

1. Job Name and Type Selection

Job Name: MyFirstJob

Job Type: Freestyle project

Description: Select an appropriate name for your job and choose the desired job type based on the nature of your project (e.g., Freestyle project, Pipeline, Multibranch Pipeline, etc.).

2. Source Code Management (SCM) Configuration

SCM Type: Perforce

Server Address: perforce.example.com:1666

Workspace Name: MyPerforceWorkspace

Description: Configure the source code management settings with Perforce by specifying the server address, workspace name, and other relevant details.

3. Build Triggers

Trigger Type: Poll SCM

Trigger Settings: Every 5 minutes

Description: Define build triggers to automate job execution based on events such as SCM changes, timed triggers, webhook triggers, or manual triggers.

4. Build Environment Configuration

Build Tools: Maven

Environment Variables: JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64

Description: Configure the build environment by specifying build tools, setting environment variables, defining build parameters, and preparing the workspace for build execution.

5. Build Steps and Post-Build Actions

Build Steps:

  • Execute shell script: mvn clean install
  • Archive artifacts: target/*.jar

Post-Build Actions:

  • Send build status notifications via email

Description: Define build steps to execute tasks such as compiling code, running tests, generating reports, and packaging artifacts. Configure post-build actions for notifications, deployments, and result reporting.

6. Additional Configuration Options

Advanced Settings:

  • Set quiet period: 30 seconds
  • Use custom workspace: /var/lib/jenkins/workspace/myjob

Permissions and Access:

  • Allow all users to build
  • Restrict project-based permissions

Description: Explore advanced configuration options for customizing build behavior, defining build triggers, setting up notifications, managing credentials, and controlling job permissions.

7. Review and Validation

Configuration Review:

  • Double-check SCM settings
  • Verify trigger configurations

Validation and Testing:

  • Run test builds on different branches
  • Check console output for errors

Description: Review the job configuration to ensure all settings are correct. Validate the job by running test builds, verifying triggers, and testing post-build actions.

8. Save and Execute

Save Job Configuration: Click "Save" to apply changes

Execute Initial Build: Trigger a manual build to test job functionality

Description: Save the job configuration to apply changes. Execute the initial build to test job functionality, validate build results, and monitor job execution.
